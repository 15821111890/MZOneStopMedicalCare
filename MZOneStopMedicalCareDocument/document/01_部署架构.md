# 01. MZOneStopMedicalCare 部署架构

> 医疗三件套（Server / Client / Business）的部署与运维细节。跨项目的基础设施约定请先读 [MZDocument/DEPLOYMENT.md](https://github.com/15821111890/MZDocument/blob/main/DEPLOYMENT.md)。

## 1. 与 MZDocument 的依赖关系

本项目**假设 `MZDocument/script/deploy_mz_gateway.python` 已在服务器跑过至少一次**：
- Nginx、MongoDB、Redis、PM2、Node 16 均已安装
- `/etc/nginx/sites-enabled/ip_default.conf` 是唯一的默认 server
- `/etc/nginx/snippets/medicine-h5.conf` 至少是空占位
- MZGateway 正在 `:3000` 运行

本仓库只负责**医疗业务自身**：Server 进程、两个 H5 构建、nginx snippet、Gateway 路由条目。**不改全局默认 server。**

## 2. 拓扑

```
Client（海外病人访问）
   ↓ http/https
Nginx :80/:443（ip_default.conf, server_name _）
   ├─ /api/medicine/*   → :3000 Gateway → :3003 medical-server
   ├─ /medicine-h5/client/      → /home/ubuntu/apps/MZOneStopMedicalCare/MZOneStopMedicalCareClient/dist/
   └─ /medicine-h5/management/  → /home/ubuntu/apps/MZOneStopMedicalCare/MZOneStopMedicalCareBusiness/dist/
```

| 组件 | 值 |
| --- | --- |
| PM2 name | `mz-medical-server` |
| 端口 | `3003` |
| 启动模式 | fork（单实例） |
| Mongo URI | `mongodb://127.0.0.1:27017/mz_medical` |
| Redis key 前缀 | `medicalAdminToken:` |
| Gateway 路由 | `/api/medicine` → `:3003`（stripPrefix: true） |
| Client base | `/medicine-h5/client/` |
| Business base | `/medicine-h5/management/` |
| Root 管理员 | `root` / `Medicine_rooter_123456`（MD5 入库；roleType=1） |

## 4. Nginx snippet 约定

`MZOneStopMedicalCareDocument/nginx/medicine.conf` 里**不能有 `server { ... }`**，只写 location：

```nginx
location /medicine-h5/client/ { ... alias ... }
location /medicine-h5/management/ { ... alias ... }
```

部署脚本把它 cp 到 `/etc/nginx/snippets/medicine-h5.conf`，`ip_default.conf` 里用 `include /etc/nginx/snippets/medicine-h5.conf;` 拉进默认 server。

**API 路由**（`/api/medicine/*`）不写在这里，由 `ip_default.conf` 的通用 `/api/` → Gateway 统一处理。

## 5. `deploy_mz_medicine_application.python` 使用

```bash
# 日常（推荐，幂等，默认跑全部步骤）
python3 MZOneStopMedicalCareDocument/script/deploy_mz_medicine_application.python

# 只跑某几步
python3 MZOneStopMedicalCareDocument/script/deploy_mz_medicine_application.python --steps server business
```

覆盖的步骤（默认顺序）：

1. `check` — Mongo/Redis 可达性探测
2. `pull` — monorepo `git pull --ff-only`
3. `server` — MZOneStopMedicalCareServer npm install + pm2 startOrReload
4. `business` — MZOneStopMedicalCareBusiness npm install + npm run build
5. `client` — MZOneStopMedicalCareClient npm install + npm run build
6. `nginx` — medicine.conf → `/etc/nginx/snippets/medicine-h5.conf` + nginx reload
7. `gateway` — Gateway `routes.json` 中的 `/api/medicine` → `:3003`（幂等）+ `pm2 reload mz-gateway`
8. `perms` — `chmod o+x /home/ubuntu /home/ubuntu/apps` + `chmod -R o+rX dist`（Nginx www-data 可读）
9. `seed_root` — 写入 `medicalAdminUsers` 的 root 账号（幂等，已存在跳过）
10. `verify` — curl `/api/medicine/health` 和两个 `/medicine-h5/...`

环境变量：

- `MZ_BASE_DIR`（默认 `/home/ubuntu/apps`）：服务器上 apps 根路径
- `MEDICAL_MONGO_URI`（默认 `mongodb://127.0.0.1:27017/mz_medical`）：覆盖医疗 DB

## 7. 账号与安全

- **独立账号体系**：与 MZServer / MZAIService 的 `members` 完全隔离，集合 `medicalAdminUsers`，token 前缀 `medicalAdminToken:`。
- 密码前端 **MD5 传输** 后入库（后端不做二次加 salt，字符串比较）。
- memberToken TTL 7 天（604800 秒）。
- Server 除 `/login`、`/logout` 外所有接口需 `memberToken` 头。

## 8. 常见排查

### 8.1 `/medicine-h5/*` 404

一般是 snippet 没同步过：跑 `--steps nginx`。

### 8.2 `/medicine-h5/*` 500（rewrite cycle）

权限问题。跑 `--steps perms`，或手动：

```bash
sudo chmod o+x /home/ubuntu /home/ubuntu/apps
sudo chmod -R o+rX /home/ubuntu/apps/MZOneStopMedicalCare/MZOneStopMedicalCareClient/dist
sudo chmod -R o+rX /home/ubuntu/apps/MZOneStopMedicalCare/MZOneStopMedicalCareBusiness/dist
sudo -u www-data stat .../dist/index.html   # 能 stat 就说明通了
```

### 8.3 `/api/medicine/health` 404

- Gateway `routes.json` 里没有 `/api/medicine`：跑 `--steps gateway`
- Nginx 没把 `/api/` 转到 Gateway：跑 MZDocument 的 `deploy_mz_gateway.python --steps sync_nginx`

### 8.4 `/api/medicine/health` 502

Gateway 活着但 Server 挂了：`pm2 logs mz-medical-server --lines 30` 看错。多数是 Mongo/Redis 连不上或 `.env` 错误。

## 9. 国际化（i18n）

- Client / Business 都接了 `vue-i18n@^9`，`legacy: false`。
- 默认语言：`localStorage`（`medicine-client-lang` / `medicine-business-lang`）优先，否则 `navigator.language` 判断 `zh-*` → `zh`，其他 → `en`。
- Locale 文件：`<project>/src/i18n/locales/{en,zh}.ts`，切换器 `<project>/src/components/LangSwitch.vue`。
- **稳定值约定**：`medicalInterest` 在前后端以英文存储和比较；`status` 是纯数字枚举。切换语言不改数据。
